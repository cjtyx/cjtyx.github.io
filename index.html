搭博客

一.GitHub+hexo搭建

步骤：

1.注册一个GitHub账号

2.创建仓库

填写用户名.github.io 切记用户名必须和左边一致

3.启用GitHub pages

点击settings进入设置，往下拉找到“GitHub Pages”

4.下载git和node.js

安装好git后桌面上任意位置右键点击出现 git gui here和git bash here则说明好了

5.安装hexo，及其组件

自选一个地方创建文件夹hexo，在此文件夹下右击鼠标，选择“Git Bash here”

输入 npm install -g hexo进行安装

使用 hexo -v ,出现下面界面说明安装成功

然后hexo init 初始化,最后的“Start blogging with Hexo！”就说明成功了

输入 npm install,安装所需组件

输入 hexo g

输入hexo s，启动本地服务器，通过访问 http://localhost:4000，体验hexo ，不过应该访问127.0.0.1:4000 才能正确访问到页面

6.部署到GitHub

1.打开你所创建的文件，找到_config.yml文件,添加如下代码（用记事本或code打开）

repo：为仓库地址，branch：分支

这儿有个特别容易出错的点，就是branch 分支，教程上branch都为master，但是GitHub更新了一些，导致如果branch为master就部署不上去，应该为branch： main ，还有一点：冒号为英文冒号，且冒号后必须有空格

2.安装一个扩展

npm install hexo-deployer-git –save

3.输入cd ~/.ssh,如果出现入下图情况，说明不存在.ssh

解决方法：输入 $ ssh-keygen -t rsa -C “xxxxxxx@qq.com“(自己邮箱) 再次如下图进行操作，出现相同文件说明成功

4.输入eval “$(ssh-agent -s)”，添加密钥到ssh-agent

5.添加SSH Key到Github，点击“settings”

填写key

6.输入ssh -T git@github.com，测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了（有一个会让你输入 yes）

7.设置你的账号信息

name和email后写自己的即可

8.部署到github hexo d 使用用户名.github.io 即可访问自己博客

9.传文件：将文件放到source/_posts 文件夹下，再部署就行

7.问题

_config.yml文件的配置：branch： main

hexo d 容易出现报错，可能是网络原因，可等等再试；可以输入git config –global http.sslVerify “false” 再试试

部署或者美化一般用 hexo clean hexo g (hexo s) hexo d

美化要在配置文件_config.yml修改，以及themes里主题下的_config.yml文件里修改

更换主题：1.下载主题文件放在themes文件下，修改配置文件下_config.yml的theme：改为主题文件的名字，再部署

访问robots.txt

flag在flagishere.txt里

robots.txt是搜索引擎蜘蛛访问网站时要查看的第一个文件，并且会根据robots.txt文件的内容来爬行网站。在某种意义上说，它的一个任务就是指导蜘蛛爬行，减少搜索引擎蜘蛛的工作量。

当搜索引擎蜘蛛访问网站时，它会首先检查该站点根目录下是否存在robots.txt文件，如果该文件存在，搜索引擎蜘蛛就会按照该文件中的内容来确定爬行的范围；如果该文件不存在，则所有的搜索引擎蜘蛛将能够访问网站上所有没有被口令保护的页面。

通常搜索引擎对网站派出的蜘蛛是有配额的，多大规模的网站放出多少蜘蛛。如果我们不配置robots文件，那么蜘蛛来到网站以后会无目的的爬行，造成的一个结果就是，需要它爬行的目录，没有爬行到，不需要爬行的，也就是我们不想被收录的内容却被爬行并放出快照。所以robots文件对于我们做网站优化来说具有很重要的影响。

svn泄露

svn仍为版本控制系统

协作开发需要版本控制系统，git和SVN，应该是目前使用最多的两个版本控制系统。在Linux环境下git，是命令行的方式，现在在Windows下，与tortoiseSVN一样，有tortoisegit，不过需要先安装git（msysgit）。

git与SVN的区别0

1.git是分布式的，SVN是集中式的

开发人员可以建立自己的分支，并在自己的分支上进行操作，减少了冲突

2.git可以在没有网络的情况下使用

从远程库clone一个到本地库，没有网络的情况下，可以将修改提交到本地库，等有网的时候再传上去。

3.git和SVN的版本控制

是通过.git和.svn文件，.git只是在本地的版本库的目录下存在，而.svn存在于每一个文件夹，当我们不需要版本控制的时候，要删除.svn很费时。

4.版本号问题

svn有明确的版本号，git对于每一个版本，都通过SHA1算法生成一个唯一标示的码，方便追溯到之前的版本。
